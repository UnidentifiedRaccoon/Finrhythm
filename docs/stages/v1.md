# v1 — stage source of truth

Этот файл описывает публично запускаемую версию продукта после успешного MVP.  
Источник истины для прогресса остаётся отдельным: `.agent/stages/v1/`.

## Как агент должен использовать этот файл

- Рассматривать MVP как prerequisite baseline.
- Не переоткрывать фундаментальные решения без явной необходимости.
- Расширять продукт без разрушения already-working core loop.
- Держать diffs малыми и milestone-driven.
- Любые публичные monetization, raffle, billing и legal-sensitive changes закрывать через explicit evidence + human gates.
- Для task execution использовать integrated proof loop внутри `$stage-launch-proof-loop` на каждый execution unit.
- Для stage-level памяти поддерживать `.agent/stages/v1/backlog.md`, `progress.md`, `decisions.md`, `risks.md`, `status.json`.

## Цель этапа

Превратить MVP в полноценный продукт для публичного запуска, где:

1. все 4 уровня прогрессии поддержаны моделью данных и UX;
2. уровни `Знаток`, `Продвинутый`, `Эксперт` имеют реальное контентное наполнение;
3. работает платная подписка;
4. магазин поддерживает несколько категорий наград и партнёрские механики;
5. reward economy юридически оформлена и операционно управляема;
6. персонализация и retention-механики заметно сильнее, чем в MVP;
7. продукт готов к public launch, нагрузке и маркетинговому росту.

## Tech baseline inherited from MVP

Если stage docs и repo state не говорят обратного, v1 наследует и развивает такой baseline:

- `apps/web` и `apps/admin` остаются на Next.js;
- `apps/api` остаётся Java Spring backend;
- PostgreSQL остаётся основным хранилищем;
- CMS продолжает хранить production-контент в PostgreSQL;
- production infra остаётся ориентированной на Yandex Cloud;
- frontend contract types продолжают генерироваться из backend OpenAPI.

## In scope

- Полное контентное расширение до всех 4 уровней.
- Расширенные типы заданий.
- Итоговые тесты по уровням.
- Сильная геймификация: лиги, weekly missions, tiered achievements.
- Полноценный магазин с несколькими категориями наград.
- Подписка и биллинг.
- Партнёрские интеграции для наград.
- Юридически корректная механика розыгрышей, если бизнес её подтверждает.
- Lifecycle marketing и retention-коммуникации.
- Personalization v2.
- CMS/Admin hardening.
- SEO, marketing site, public launch operations.
- Performance, accessibility, security hardening.

## Out of scope

- AI tutor.
- Social graph с полноценной социальной лентой.
- Team marathons.
- White-label / multi-tenant B2B.
- Полная локализация multi-region.
- Marketplace 2.0 с автоматическими settlement pipeline.
- Enterprise SSO / SCIM.
- Adaptive learning engine на уровне онлайн-оптимизации difficulty per item.

## Human gates

- Юридическое утверждение billing/raffle/partner terms.
- Подтверждение pricing strategy и paywall boundaries.
- Финальное утверждение public brand/campaign copy.
- Финальная экспертная верификация расширенного финансового контента.
- Реальный operational onboarding партнёров и каталога наград.
- Готовность support/comms-команды к public launch.

## Exit gates

v1 считается закрытой только если:

- Пользователь проходит согласованный public funnel от landing до first learning win.
- Все 4 уровня видимы в интерфейсе и минимум по одному полноценному learning path есть для каждого уровня.
- Есть итоговые level-end assessments.
- Работает платная подписка минимум в одном тарифном сценарии.
- Магазин поддерживает расширенный каталог, статусы, квоты и partner-backed rewards.
- Если включены розыгрыши, у них есть audit trail, eligibility rules и human-approved legal package.
- Lifecycle flows и retention analytics работают.
- Admin/CMS покрывают реальную повседневную операционку.
- Продукт выдерживает planned public launch readiness audit.
- Все blocking issues из pre-launch hardening закрыты.

## Правила stage execution

- Любой monetization-related unit сначала замораживается как spec с явными non-goals.
- Если задача затрагивает payment/billing/legal/store/redemption одновременно, дробить обязательно.
- Перед public launch каждый крупный milestone обязан иметь own verification pack.
- При любых спорных trade-off первичны: trust, legal clarity, user comprehension, data integrity.
