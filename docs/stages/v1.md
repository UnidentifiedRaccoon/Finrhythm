# v1 — stage source of truth

Этот файл описывает stage после успешного B2B-first MVP pilot. Источник истины для прогресса остаётся отдельным: `.agent/stages/v1/`.

v1 does not automatically mean public B2C launch or in-app subscription. Public launch, B2C monetization and subscription are optional product tracks that require explicit human-approved commercial decision before implementation.

## Как агент должен использовать этот файл

- Рассматривать MVP as prerequisite B2B pilot baseline.
- Читать `docs/product/b2b-mvp/lemanapro/product-foundation-v1.md` and `docs/product/b2b-mvp/lemanapro/learning-methodology-v0.2.md` before spec freeze unless a newer product foundation/methodology supersedes them.
- Не переоткрывать фундаментальные MVP decisions, including `C1–C10`, `N1–N7`, `SA1–SA3`, `R1–R6`, aggregate-only HR learning data and no-photo/no-doc/no-exact-sum defaults, без explicit product/stage doc update.
- Расширять продукт для repeatable corporate rollout: more content, stronger reporting, operations, security, lifecycle and admin maturity.
- Любые monetization, partner, raffle, customer-specific reporting and legal-sensitive changes закрывать через evidence + human gates.
- Для task execution использовать integrated proof loop внутри `$stage-launch-proof-loop` на каждый execution unit.
- Для stage-level памяти поддерживать `.agent/stages/v1/backlog.md`, `progress.md`, `decisions.md`, `risks.md`, `status.json`.

## Цель этапа

Превратить MVP-пилот в воспроизводимый B2B rollout product, где:

1. корпоративные волны можно запускать повторяемо without forking product;
2. контент и диагностика глубже покрывают потребности сотрудников;
3. HR/wellbeing sponsor получает stronger analytics, pilot comparison and scale-readiness reports;
4. privacy, support, merch operations and auditability достаточно зрелые для нескольких корпоративных волн;
5. lifecycle communications and retention mechanics improve return behavior;
6. optional B2C/public/subscription path can be evaluated without becoming implicit scope.

## Baseline inherited from MVP

- `apps/web` and `apps/admin` remain Next.js + React.
- `apps/api` remains Spring Boot + Java + Maven + PostgreSQL.
- CMS/PostgreSQL remains production content source of truth.
- Backend OpenAPI remains integration contract for frontend/admin.
- B2B pilot concepts from MVP remain canonical: `tenant`, `cohort/wave`, invite code, aggregated HR reports, points ledger, merch operations and privacy threshold.
- MVP learning methodology v0.2 remains canonical for the first curriculum/routing model: `C1–C10`, `N1–N7`, optional `Z1/Z4/Z9`, `SA1–SA3`, `R1–R6`, content approve-flow and sensitive learning-data boundaries.
- Employee-facing UI remains neutral-branded unless a human-approved branding stage changes this.

## In scope

- Content expansion beyond the first 7-step track.
- Diagnostic refinement, stronger routing and review/remediation mechanics.
- Repeatable cohort/wave operations for additional corporate launches.
- Stronger HR dashboards, pilot comparison and exportable reports.
- Lifecycle messaging: email/in-app reminders, HR communication kit, reactivation flows.
- Store and merch operations hardening: inventory, quotas, exports, manual adjustments, audit logs.
- Admin/CMS hardening for content, rewards, support and reports.
- Privacy/security/accessibility/performance hardening for corporate rollout.
- Support handbook, operational runbooks and rollout checklists.
- Optional commercial discovery docs for B2C/subscription or broader B2B packaging, with human gate before code implementation.

## Out of scope

- Mandatory public B2C launch.
- Mandatory in-app subscription or payment system.
- Partner financial product marketplace.
- Enterprise SSO/SCIM as default requirement.
- Self-service multi-tenant enterprise platform.
- Full white-label customer branding in employee UI.
- AI tutor or autonomous AI planning.
- Open social graph, public personal leaderboards and team competitions.
- Automated marketplace settlement pipeline.
- Multi-region localization rollout.

## Human gates

- Commercial decision to start B2C/public/subscription work.
- Pricing, paywall, billing policy and contract model.
- Legal/privacy review for expanded reporting, exports and customer-specific analytics.
- Final expert review for expanded financial content and diagnostics.
- Real rewards, partners, inventory and fulfillment operations.
- Customer branding, external case-study copy and public campaign claims.

## Exit gates

v1 is complete only if:

- Additional corporate waves can be prepared, launched and reported with documented runbooks.
- Content coverage extends beyond MVP while keeping methodologist/fact-check statuses explicit.
- Diagnostics and personalized route are more useful and still explainable.
- HR/sponsor reports compare cohorts/waves without exposing sensitive personal answers.
- Store/merch operations are auditable and manageable beyond the first pilot.
- Support/comms team has handbook, templates and triage flow.
- Admin/CMS covers everyday operations without ad hoc database edits.
- Performance/accessibility/security checks are sufficient for planned corporate rollout.
- Optional B2C/subscription work is either explicitly out of scope or captured as human-approved separate stage/slice.
- Latest audit has no blocking FAIL/UNKNOWN outside explicit human gates.

## Правила stage execution

- Any unit touching privacy/reporting/rewards/support/admin exports must freeze human gates and evidence targets first.
- If a task touches code, API, admin, analytics and docs together, split it before implementation.
- B2C/subscription/public-launch work cannot start from this file alone; it needs an explicit product decision artifact.
- При любых спорных trade-off первичны: trust, legal clarity, user comprehension, data integrity, sponsor usefulness.

## V1-01. Curriculum and diagnostics expansion

**Goal:** broaden educational value after the first 7-step MVP track without reopening MVP methodology unless pilot evidence justifies a product/stage update.

**Dependencies:** MVP complete
**Parallelism:** medium with content/admin split

### Execution units

- V1-01.01 [agent+human] Expand competency matrix and topic tree from `C1–C10` after MVP evidence.
- V1-01.02 [agent+human] Add new lessons, quizzes and practical assignments for priority topics.
- V1-01.03 [agent] Implement richer question types where needed.
- V1-01.04 [agent] Implement review/remediation queue for weak topics.
- V1-01.05 [agent+human] Update glossary, tone-control rules and fact-check workflow.

### Acceptance criteria

- Expanded content is structured, reviewed and publishable through CMS.
- Diagnostics/routing remain explainable and non-shaming.
- Refinements preserve aggregate-only HR reporting, no personal weak-zone disclosure and no exact-sum/photo/document requirement by default.
- New exercises do not break analytics, progress or reporting.

### Evidence required

- Published content list.
- Review statuses.
- Diagnostic/routing tests.
- Preview screenshots.
