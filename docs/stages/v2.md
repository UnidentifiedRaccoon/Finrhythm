# v2 — stage source of truth

Этот файл описывает stage после успешного public launch.  
Здесь фокус не на “доделать базу”, а на усилении differentiators: adaptive learning, AI layer, social loop, marketplace automation, B2B/B2B2C и platform maturity.

## Как агент должен использовать этот файл

- Рассматривать v1 как prerequisite baseline.
- Не переписывать working surfaces без веской причины и без migration path.
- Новые high-risk capabilities вводить через isolated milestones, evaluations и guardrails.
- Любые AI-facing и social-facing changes считать risk-heavy by default.
- Использовать integrated proof loop внутри `$stage-launch-proof-loop` на каждый execution unit.
- Для stage-level state поддерживать `.agent/stages/v2/`.

## Цель этапа

Создать версию продукта, которую сложнее быстро скопировать конкурентам, за счёт:

1. глубокой персонализации и adaptive learning;
2. AI tutor / AI-assisted support and planning;
3. social graph и командных активностей;
4. сезонов, событий и более зрелой game economy;
5. partner marketplace automation;
6. B2B/B2B2C и tenant-aware возможностей;
7. localization / multi-region readiness;
8. зрелой data/experimentation/governance платформы.

## Platform baseline inherited from v1

Если нет явного нового архитектурного решения, v2 развивается поверх такого baseline:

- monorepo c `apps/web`, `apps/admin`, `apps/api`;
- Next.js для frontend surfaces;
- Java Spring для backend platform;
- PostgreSQL + object storage + optional Redis;
- Yandex Cloud как базовая target-инфраструктура;
- OpenAPI-generated TS contracts между backend и frontend;
- CMS/PostgreSQL как content source of truth.

## In scope

- Adaptive learning engine.
- AI tutor, AI explanations, AI-assisted practice.
- AI-assisted support triage and individual plan drafts.
- Friends/follows, team marathons, social comparisons with privacy controls.
- Seasons, events, collectible and cosmetic reward layers.
- Partner marketplace automation.
- B2B/B2B2C foundations.
- Localization framework and regional switches.
- Experimentation platform.
- Infra, cost, reliability and governance maturity.

## Out of scope

- Полноценная open community/social feed с user-generated long-form content.
- Unbounded AI autonomy without evals and safety gates.
- International launch in many jurisdictions simultaneously.
- Marketplace for regulated financial advice without legal framework.
- Fully autonomous operations without human review paths.
- Any reward mechanic that makes points directly redeemable for cash.

## Human gates

- AI safety policy approval.
- Legal/privacy review for social features and AI data use.
- Partner contracts and marketplace policy approvals.
- B2B contract/security/compliance review.
- Regional legal approval for localization rollout.
- Public communication strategy for AI features.

## Exit gates

v2 можно считать завершённой только если:

- adaptive learning улучшает routing/review quality и это видно в метриках;
- AI tutor и AI-assisted features имеют eval suite, safety policy и monitoring;
- social features работают с приватностью, moderation и abuse controls;
- season/event systems не ломают points economy;
- marketplace automation не ломает ledger integrity;
- B2B/B2B2C baseline позволяет запускать пилоты;
- localization framework позволяет добавлять новые языки без архитектурного слома;
- experimentation/data platform поддерживает product decisions;
- infra/reliability/governance на уровне scale stage.

## Общие правила v2

- AI feature нельзя считать complete без evals, fallback behavior и human escalation.
- Social feature нельзя считать complete без privacy settings, moderation path и abuse monitoring.
- Marketplace feature нельзя считать complete без audit trail, idempotency и settlement safety.
- B2B feature нельзя считать complete без tenant isolation assumptions.
- Localization feature нельзя считать complete без fallback strategy.

## V2-01. Adaptive learning engine

**Goal:** превратить маршрут и review в динамическую систему, а не набор rule-based хаков.

**Dependencies:** v1 personalization and analytics baseline  
**Parallelism:** low-medium

### Execution units
- V2-01.01 [agent+human] Зафиксировать mastery model по навыкам и темам
- V2-01.02 [agent] Реализовать adaptive difficulty rules
- V2-01.03 [agent] Реализовать session planner и smart review queue
- V2-01.04 [agent] Реализовать next-best-session generation
- V2-01.05 [agent] Реализовать personalized challenge selection
- V2-01.06 [agent] Реализовать evaluation dashboards по learning lift / routing quality
- V2-01.07 [agent] Реализовать rollback / fallback path к personalization v1

### Acceptance criteria
- Adaptive layer влияет на порядок и сложность обучения, а не только на copy.
- Есть measurable uplift or at least controlled experiment hooks.
- Fallback к прежним правилам возможен.
- Пользователь не теряет понимание, почему ему предлагают этот маршрут.

### Evidence required
- Model docs.
- Routing demos.
- Experiment dashboards.
- Fallback tests.
